C#静态类 和 C++静态实例 的构造区别

2014-08-29 C/C++►杂项 C#, C/C++ 0 评论 字数统计: 723(字) 阅读时长: 3(分)

在调用DiskQuota时遇到了异常:  "****.DiskQuota”的类型初始值设定项引发异常。

这让我去仔细看了下MSDN上C#的静态类的解释, 有这么一句:

“… However, it is guaranteed to be loaded and to have its fields initialized and its static constructor called before the class is referenced for the first time in your program.  …”

原来C#的静态类只保证在首次调用前初始化, 这里的初始化包括静态成员赋值以及调用静态构造函数.这样程序就必须保证在首次调用前捕获可能发生的异常,而且如果首次初始化发生异常,那么后续的和该静态类有关的成员以及成员函数调用都会有异常:

两种形式的静态实例都会在main函数之前抛出异常, 而且这个异常无法捕获, 只能在异常触发后捕获到SIGABRT, 在main函数前捕获这个信号可以这样做:

Linux下可能看到这样的输出:

这样也只是捕获到信号,仅此而已,无法处理异常,因此C++要求更严格,存在异常的静态实例构造的程序连启动都谈不上.

可见各有各的好处:

C#实现了一个纯静态的单件模式,而且允许捕获其异常(如果需要实现非静态的单件实例可以参考这里);

C++要求更安全静态初始化过程,同样也可以用于单件模式.
